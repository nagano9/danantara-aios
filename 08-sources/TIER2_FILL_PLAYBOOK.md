# Tier 2 Fill Playbook

This playbook explains how to turn the Tier 2 source-layer scaffold into a set
of authenticated Danantara policy and delegation documents.

## Purpose

- fill the most blocking source gaps first
- keep each Tier 2 document tied to a real decision
- make every document traceable by owner, effective date, and retrieval location
- preserve an audit trail from placeholder to authenticated source

## Fill order

Follow the order that blocks the most decisions:

1. Badan Pelaksana delegation of authority matrix
2. DIM Investment Committee charter
3. Investment policy statement
4. Conflict-of-interest and recusal policy
5. Data classification and approved-AI-environment policy
6. Entity model map

Then proceed to:

7. Dewan Pengawas charter
8. DAM active-ownership and transformation policy
9. DDMF development-finance and additionality policy
10. Valuation policy and model-approval standard
11. Procurement and integrity due-diligence standard
12. Whistleblowing and escalation procedure

## Minimum evidence for promotion

Before a placeholder can be promoted, the source must have:

- owner
- title
- version
- effective date
- approval body
- document location
- classification
- retrieval path that an independent reviewer can follow

If a document is amended, cite the amended instrument, not the older draft.

## Promotion workflow

1. Collect the authenticated document.
2. Record the metadata in the Tier 2 template.
3. Link the document to the backlog items it unlocks.
4. Remove or retire the placeholder entry.
5. Update the index and the source register.
6. Re-run the Tier 2 scaffold validator.

## Quality checks

- every promoted document must be independently retrievable
- no placeholder may be treated as a source
- the source register must stay aligned with the working folder
- the backlog must show which decisions the source unlocks
