# Repo Confidentiality Leakage Source Map

Use this file as the repo-native entry point for `confidentiality-leakage-check`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved assurance artifacts and controlled templates.
3. Classification, disclosure, leakage, and access-control rules.
4. Skill-local trigger cases and reference notes.
5. Prior review outputs only as context.

If the question is about classification, use `data-sovereignty-assessor`. If it is about narrative or external wording, use `transparency-disclosure-gate`.

## Usage notes

- Use the source map before flagging a leak.
- Keep classification, access, and audience separate.
- Do not conflate openness with leakage.
