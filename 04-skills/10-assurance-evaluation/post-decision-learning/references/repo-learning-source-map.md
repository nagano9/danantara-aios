# Repo Learning Source Map

Use this file as the repo-native entry point for `post-decision-learning`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Decision record, prediction, outcome evidence, and review notes.
4. Pattern evidence from prior decisions and regression cases.
5. Prior lessons only as context.

If the prediction was written after the outcome, or if the outcome is being used to rewrite the prediction, the record is invalid for learning.

## Usage notes

- Use the source map before settling a prediction.
- Keep process and outcome separate.
- If the prediction is missing, return `unlearnable` and do not invent a lesson.
- Route the decision lesson, skill defect, and workbook gap to different sinks.
