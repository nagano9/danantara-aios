# Repo Public Service Obligation Source Map

Use this file as the repo-native entry point for `public-service-obligation-analyzer`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Public service obligation logic, subsidy treatment, and measurable service rules.
4. Skill-local trigger cases and reference notes.
5. Prior PSO outputs only as context.

If the question is about impact or financing, use `development-impact-assessor` or `project-finance-structurer`. If it is about capital support, use `viability-gap-support-assessor`.

## Usage notes

- Use the source map before calling a cost support a service obligation.
- Keep service level, subsidy, and delivery model separate.
- Do not hide an operating deficit inside a mandate label.
