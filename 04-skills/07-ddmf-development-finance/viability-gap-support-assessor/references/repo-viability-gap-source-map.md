# Repo Viability Gap Source Map

Use this file as the repo-native entry point for `viability-gap-support-assessor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Viability gap, subsidy logic, and support conditionality rules.
4. Skill-local trigger cases and reference notes.
5. Prior support outputs only as context.

If the question is about financing structure, use `project-finance-structurer`. If it is about public service need, use `public-service-obligation-analyzer`.

## Usage notes

- Use the source map before treating a gap as justification for support.
- Keep viability, subsidy, and policy purpose separate.
- Do not assume every gap should be filled.
