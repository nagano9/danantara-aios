# Repo Bankability Source Map

Use this file as the repo-native entry point for `bankability-gap-analyzer`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Bankability, feasibility, coverage, and risk allocation rules.
4. Skill-local trigger cases and reference notes.
5. Prior bankability outputs only as context.

If the question is about financing design, use `project-finance-structurer`. If it is about public value, use `development-impact-assessor`.

## Usage notes

- Use the source map before stating a project is bankable.
- Keep bankability separate from desirability.
- Do not treat a funding expression as proof of feasibility.
