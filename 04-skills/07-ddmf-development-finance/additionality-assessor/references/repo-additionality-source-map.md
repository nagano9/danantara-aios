# Repo Additionality Source Map

Use this file as the repo-native entry point for `additionality-assessor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Additionality logic, baseline, counterfactual, and catalytic value rules.
4. Skill-local trigger cases and reference notes.
5. Prior additionality outputs only as context.

If the question is about bankability or financing structure, use `bankability-gap-analyzer` or `project-finance-structurer`. If it is about public outcome, use `development-impact-assessor`.

## Usage notes

- Use the source map before claiming an intervention is additional.
- Keep baseline, counterfactual, and attribution distinct.
- Do not confuse funding with additionality.
