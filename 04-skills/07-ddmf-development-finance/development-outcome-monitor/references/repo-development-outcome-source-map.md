# Repo Development Outcome Source Map

Use this file as the repo-native entry point for `development-outcome-monitor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Outcome tracking, milestone monitoring, and attribution rules.
4. Skill-local trigger cases and reference notes.
5. Prior outcome outputs only as context.

If the question is about early impact design, use `development-impact-assessor`. If it is about delivery progress, use `project-delivery-monitor`.

## Usage notes

- Use the source map before claiming an outcome is on track.
- Keep output, outcome, and attribution separate.
- Do not turn milestone completion into impact proof.
