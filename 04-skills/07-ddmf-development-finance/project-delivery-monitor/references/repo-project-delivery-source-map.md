# Repo Project Delivery Source Map

Use this file as the repo-native entry point for `project-delivery-monitor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Delivery milestones, stage gates, dependencies, and exception rules.
4. Skill-local trigger cases and reference notes.
5. Prior delivery outputs only as context.

If the question is about readiness to start, use `project-readiness-gate`. If it is about financing conditions, use `project-finance-structurer`.

## Usage notes

- Use the source map before saying a project is on schedule.
- Keep milestone, dependency, and actual delivery distinct.
- Do not treat a plan date as a delivered date.
