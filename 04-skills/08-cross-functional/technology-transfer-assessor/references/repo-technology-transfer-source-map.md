# Repo Technology Transfer Source Map

Use this file as the repo-native entry point for `technology-transfer-assessor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Know-how, IP, training, supplier, standards, and operating-independence rules.
4. Skill-local trigger cases and reference notes.
5. Prior transfer outputs only as context.

If the question is about procurement or JV design, use `procurement-commercial-strategy`. If it is about capability building, use `industrialization-hilirisasi-analyzer`.

## Usage notes

- Use the source map before claiming transfer is enduring.
- Keep IP access, training, supplier development, and autonomy separate.
- Do not confuse delivery of equipment with transfer of capability.
