# Repo Cybersecurity By Design Source Map

Use this file as the repo-native entry point for `cybersecurity-by-design`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Security architecture, threat, control, and least-privilege rules.
4. Skill-local trigger cases and reference notes.
5. Prior security outputs only as context.

If the question is about data handling, use `data-sovereignty-assessor`. If it is about risk aggregation, use `enterprise-risk-aggregator`.

## Usage notes

- Use the source map before calling something secure by default.
- Keep architecture, threat model, and control evidence separate.
- Do not confuse compliance paperwork with actual security.
