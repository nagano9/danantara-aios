# Repo Stakeholder Communications Source Map

Use this file as the repo-native entry point for `stakeholder-communications-planner`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Stakeholder mapping, disclosure, message sequencing, and escalation rules.
4. Skill-local trigger cases and reference notes.
5. Prior communications outputs only as context.

If the question is about disclosure gating, use `transparency-disclosure-gate`. If it is about tone, use `vocabulary-and-narrative-guard`.

## Usage notes

- Use the source map before mapping a message to an audience.
- Keep interests, evidence, commitments, and timing separate.
- Do not promise what the decision path cannot support.
