# Repo Operational Excellence Source Map

Use this file as the repo-native entry point for `operational-excellence-diagnostic`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Operational performance, process, capability, and control rules.
4. Skill-local trigger cases and reference notes.
5. Prior diagnostic outputs only as context.

If the question is about financial results, use `financial-performance-diagnostic`. If it is about turnaround, use `turnaround-triage`.

## Usage notes

- Use the source map before naming the root cause.
- Keep symptom, process defect, and control failure separate.
- Do not confuse a busy process with an effective one.
