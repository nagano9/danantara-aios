# Repo Restructuring Options Source Map

Use this file as the repo-native entry point for `restructuring-options-designer`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Restructuring, recapitalization, operational fix, and ownership rules.
4. Skill-local trigger cases and reference notes.
5. Prior restructuring outputs only as context.

If the question is about distress diagnosis, use `turnaround-triage`. If it is about capital, use `capital-allocation-reviewer`.

## Usage notes

- Use the source map before selecting a restructuring path.
- Keep operational, financial, and ownership fixes separate.
- Do not force one tool to solve a structural problem.
