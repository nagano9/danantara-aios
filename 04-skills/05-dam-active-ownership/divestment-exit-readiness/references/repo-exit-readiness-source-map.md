# Repo Divestment Exit Readiness Source Map

Use this file as the repo-native entry point for `divestment-exit-readiness`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Exit readiness, marketability, separation, and disposal logic.
4. Skill-local trigger cases and reference notes.
5. Prior exit outputs only as context.

If the question is about asset exit design, use `underperforming-asset-playbook`. If it is about capital redeployment, use `capital-redeployment-planner`.

## Usage notes

- Use the source map before describing a company as exit-ready.
- Keep marketability, timing, and governance cleanly separated.
- Do not confuse tiredness with readiness to sell.
