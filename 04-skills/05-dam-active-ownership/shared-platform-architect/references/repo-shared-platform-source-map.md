# Repo Shared Platform Source Map

Use this file as the repo-native entry point for `shared-platform-architect`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Shared-services, platform, operating-model, and control rules.
4. Skill-local trigger cases and reference notes.
5. Prior platform outputs only as context.

If the question is about digital value creation, use `enterprise-digital-synergy`. If it is about governance or risk, use `enterprise-risk-aggregator`.

## Usage notes

- Use the source map before standardizing a platform.
- Keep architecture, governance, and operating model separate.
- Do not call a platform shared if its controls are not.
