# Repo Management Accountability Source Map

Use this file as the repo-native entry point for `management-accountability-contract`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Accountability design, role clarity, and management obligations.
4. Skill-local trigger cases and reference notes.
5. Prior accountability outputs only as context.

If the question is about CEO challenge, use `ceo-transformation-challenger`. If it is about change implementation, use `transformation-agenda-orchestrator`.

## Usage notes

- Use the source map before defining who is accountable for what.
- Keep role, KPI, and consequence distinct.
- Do not make accountability vague just to keep peace.
