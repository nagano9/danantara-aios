# Repo Post Merger Integration Source Map

Use this file as the repo-native entry point for `post-merger-integration-planner`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Day-1 continuity, integration, controls, and value-capture rules.
4. Skill-local trigger cases and reference notes.
5. Prior integration outputs only as context.

If the question is about the deal itself, use `merger-consolidation-case`. If it is about change sequencing, use `transformation-agenda-orchestrator`.

## Usage notes

- Use the source map before going from deal thesis to Day-1 plan.
- Keep systems, people, controls, and brand separate.
- Do not assume the merger closes itself after signing.
