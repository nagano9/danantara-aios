# Repo Merger Consolidation Source Map

Use this file as the repo-native entry point for `merger-consolidation-case`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Merger logic, consolidation options, and integration consequences.
4. Skill-local trigger cases and reference notes.
5. Prior merger outputs only as context.

If the question is about integration planning, use `post-merger-integration-planner`. If it is about operational benefit, use `enterprise-digital-synergy`.

## Usage notes

- Use the source map before calling consolidation a value solution.
- Keep transaction case, integration case, and operating case separate.
- Do not undercount integration cost because the thesis looks attractive.
