# Repo Digital Infrastructure Sector Source Map

Use this file as the repo-native entry point for `digital-infrastructure-sector-pack`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Sector market, technology, operating, financing, and sovereignty rules.
4. Skill-local trigger cases and reference notes.
5. Prior sector-pack outputs only as context.

If the question is about competitiveness or industrial capability, use `national-champion-competitiveness`. If it is about digital risk, use `cybersecurity-by-design`.

## Usage notes

- Use the source map before giving a sector view.
- Keep market, policy, architecture, and risk separate.
- Do not confuse digital presence with digital advantage.
