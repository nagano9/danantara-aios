# Repo National Resilience Source Map

Use this file as the repo-native entry point for `national-resilience-assessor`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Resilience, continuity, shock absorption, and systemic risk rules.
4. Skill-local trigger cases and reference notes.
5. Prior resilience outputs only as context.

If the question is about downside or loss, use `prudence-and-downside-gate`. If it is about continuity in execution, use `workflow-state-manager`.

## Usage notes

- Use the source map before calling a system resilient.
- Keep robustness, redundancy, and survivability separate.
- Do not equate continuity with invulnerability.
