# Repo Technical Source Map

Use this file as the repo-native entry point for `technical-operational-due-diligence`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Technical, operational, resilience, and assurance protocols.
4. Owner-certified internal records and primary transaction or project documents.
5. Prior diligence only as context.

If the question is whether the pack is ready for decision, use `decision-quality-gate`. If the question is whether the asset should be monitored after closing, use `post-investment-monitor`.

- Keep diligence separate from mandate and approval questions.
- Do not convert operational complexity into optimism.
