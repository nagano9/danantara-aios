# Repo Legal-Tax Source Map

Use this file as the repo-native entry point for `legal-tax-due-diligence`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Legal, tax, disclosure, and approval protocols.
4. Owner-certified internal records and primary transaction or project documents.
5. Prior diligence only as context.

If the question is whether the approval is valid, use `human-approval-orchestrator`. If the question is whether there is a conflict, use `conflict-of-interest-gate`.

## Usage notes

- Use the source map before summarizing legal or tax diligence.
- Keep legal/tax findings separate from approval and quality findings.
- Do not use legal nuance to invent authority.
