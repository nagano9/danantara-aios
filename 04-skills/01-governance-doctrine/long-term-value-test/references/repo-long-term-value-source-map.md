# Repo Long-Term Value Source Map

Use this file as the repo-native entry point for `long-term-value-test`. It is an index, not an authority by itself.

## Source order

1. Law, regulation, official decision, and formal mandate.
2. Approved Danantara governance artifacts and controlled templates.
3. Long-term value, downside, impact, and decision-quality protocols.
4. Owner-certified internal records and primary transaction or project documents.
5. Prior value assessments only as context.

If the question is about commercial return versus national consequence, use `dual-mandate-balancer`. If the question is whether the pack is ready for decision, use `decision-quality-gate`.

## Usage notes

- Use the source map before judging long-term value.
- Keep long-term value separate from mandate and approval questions.
- Do not compress strategic durability into a single score.
