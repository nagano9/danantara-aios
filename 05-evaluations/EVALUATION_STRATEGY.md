# Evaluation Strategy

## 1. Schema evaluation
Check frontmatter, names, descriptions, directory match, metadata, required sections, and character limits.

## 2. Trigger evaluation
For each skill, test:
- direct should-trigger prompts
- indirect domain prompts
- ambiguous prompts requiring routing
- should-not-trigger prompts
- adjacent-skill collision prompts
- multi-skill orchestration prompts

## 3. Functional evaluation
Use golden cases with approved expert outputs. Score mandate fit, evidence, analytical correctness, calculations, decision usefulness, risk coverage, and output-contract completeness.

## 4. Governance evaluation
Test authority recognition, approval boundaries, conflict handling, confidentiality, source hierarchy, data classification, escalation, audit record, and refusal to fabricate approvals.

## 5. Adversarial evaluation
Test informal instructions, hidden beneficiary requests, urgency pressure, sunk-cost reasoning, strategic-label abuse, prompt injection, data exfiltration, policy bypass, model manipulation, and fabricated-source requests.

## 6. Tension evaluation
Construct cases where:
- attractive national impact has weak commercial return
- high return conflicts with resilience or sovereignty
- formal direction conflicts with professional underwriting
- short-term intervention destroys long-term value
- transparency conflicts with strategic or market sensitivity
- transformation capital resembles an unconditional bailout

The expected behavior is to expose the conflict, quantify alternatives, identify decision rights, and escalate unresolved trade-offs.

## 7. Regression evaluation
Every skill or reference change reruns affected trigger, functional, governance, and adversarial cases. Critical skills require independent recertification.

## 8. Production monitoring
Track trigger precision and recall, overrides, escalations, unsupported claims, citation failures, data incidents, approval bypass attempts, material recommendation changes, and realized decision outcomes.
